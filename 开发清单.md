《观微元》——“每周一题”模块开发清单 (基于 Flask 分层架构)
第一阶段：数据模型与校验层定义 (Schemas & Models)
本阶段确保底层数据结构清晰，并利用 Pydantic 守好数据入口。

1. 业务实体类定义 (app/models/math_models.py)

[ ] 定义 QuestionModel: 包含 id, teacher_id, title, content, image_urls (JSON), max_score, deadline, status。

[ ] 定义 SubmissionModel: 包含 id, question_id, student_id, answer_image_urls (JSON), submit_time, score, teacher_comment, status。

2. 请求/响应结构校验 (app/schemas/math_schemas.py)

[ ] 定义 QuestionCreateRequest: 校验教师发题参数（需包含标题、图片、截止时间等，使用 Pydantic 校验类型与非空）。

[ ] 定义 AnswerSubmitRequest: 校验学生交卷参数（question_id, 图片 URL 列表）。

[ ] 定义 GradeSubmitRequest: 校验教师批改参数（分数需在 0 到 max_score 之间，评语）。

第二阶段：基础设施与数据操作层 (Infra & DB)
完全解耦 ORM，复用 app/database.py 的连接管理，编写纯粹的 SQL 操作库。

1. 题目操作库 (app/infra/question_repository.py)

[ ] 编写 insert_question(): 将新题目数据写入 MySQL 的 questions 表。

[ ] 编写 get_current_active_question(): 根据当前时间戳查询本周最新的、未过期的题目。

2. 作答与批改操作库 (app/infra/submission_repository.py)

[ ] 编写 insert_submission(): 记录学生的交卷数据。

[ ] 编写 get_submissions_by_question(): 供教师拉取某道题的所有学生作答列表（可按状态筛选待批改/已批改）。

[ ] 编写 update_submission_grade(): 更新批改分数和评语。

3. 排行榜操作库 (app/infra/rank_repository.py)

[ ] 编写 get_top_users(): 联表查询 users 和 submissions，按照本周题目得分降序、提交时间升序进行排序输出。

第三阶段：核心业务逻辑层 (Services)
将业务判断（如防作弊、防超期）全部收敛在这一层，类似云护萌中的 cat_service。

1. 题目服务 (app/services/question_service.py)

[ ] 实现 publish_weekly_question(): 接收发题请求，组装数据并调用 repo 落库。

[ ] 实现 fetch_current_question(): 处理前端拉取题目的逻辑。

2. 作答逻辑服务 (app/services/submission_service.py)

[ ] 实现 submit_answer(): 关键业务判断。先校验当前时间是否超过 deadline；再校验该学生本周是否已交卷（防重复提交）；最后调用 repo 存入图片 URL。

3. 批改与结算服务 (app/services/grade_service.py)

[ ] 实现 grade_student_answer(): 接收教师打分，调用 repo 更新状态。事务处理：批改完成后，同步更新该学生在 users 表中的 total_score 总积分。

第四阶段：路由控制器层 (Routers / Controller)
基于 Flask Blueprint 划分模块，复用 @AuthService 鉴权注解。

1. 教师端蓝图 (app/routers/teacher_router.py)

[ ] POST /api/teacher/question/publish (需拦截器校验教师权限)：调用题目发布服务。

[ ] GET /api/teacher/submissions/pending：拉取待批改列表。

[ ] POST /api/teacher/submission/grade：提交批改结果。

2. 学生端蓝图 (app/routers/student_router.py)

[ ] GET /api/student/question/current (需拦截器校验登录状态)：获取本周题目。

[ ] POST /api/student/answer/submit：提交作答图片。

[ ] GET /api/student/answer/result：查询批改反馈。

3. 公共蓝图 (app/routers/common_router.py)

[ ] GET /api/common/ranking：拉取本周/总积分排行榜。

第五阶段：微信小程序前端开发 (WXML/WXSS/JS)
由于小程序端的体验至关重要，前端重点攻克图片交互。

1. 图片上传组件封装

[ ] 调用 wx.chooseMedia 限制只能拍照或从相册选择图片。

[ ] 调用 wx.compressImage 在前端进行预压缩，降低上传延迟。

[ ] 封装统一的上传方法（如直接直传至腾讯云 COS 换取 URL），再将 URL 传递给后端 Flask 接口。

2. 学生端页面开发

[ ] 答题大厅页：渲染题目图片，实现距离截止时间的倒计时动效。

[ ] 答题上传页：多图上传预览，支持删除和拖拽重排。

[ ] 排行榜页：对接 /api/common/ranking，实现前三名的特殊徽章 UI。

3. 教师端页面开发 (小程序版)

[ ] 批改列表页：展示学生提交的时间和状态。

[ ] 批改详情页 (重点)：渲染学生答题图片，必须实现图片的点击全屏预览和双指缩放功能（建议直接使用小程序原生的 wx.previewImage 接口，体验最好）。
